# AI Contract Guard

*[English version](README.md)*

Контрактное тестирование для интеграций с LLM — недостающая страховочная сетка между вашими промптами и продакшеном. Опишите **контракт** — каким должен быть корректный ответ модели (JSON-схема, обязательные поля, запрещённые паттерны, тон, бюджет по стоимости/латентности), а AI Contract Guard будет ловить тихий дрейф поведения при незаметном обновлении модели провайдером, автоматически генерировать регрессионные тесты из реальных сбоев и блокировать деплой в CI/CD до того, как плохой промпт или смена модели дойдут до пользователей.

## Зачем

В отличие от обычного API, вывод LLM не гарантированно стабилен между версиями модели, температурой или обновлениями на стороне провайдера. Команды обычно замечают «ИИ стал хуже» только после жалоб пользователей. AI Contract Guard заимствует идею **контрактного тестирования** (как Pact для REST API) и применяет её к ИИ: ваш контракт — источник истины, и каждый ответ — в разработке или в проде — проверяется против него, а история сохраняется, чтобы дрейф со временем был виден, а не оставался догадкой.

## Основные идеи

1. **Контракты** — YAML/JSON-документы, описывающие, как выглядит *корректный* ответ для конкретного промпта/сценария: JSON-схема, обязательные поля, запрещённые regex-паттерны, максимум токенов, латентности, стоимости.
2. **Валидатор** — проверяет один ответ против контракта, возвращает структурированный `ValidationResult` с результатом по каждому правилу.
3. **Хранилище** — каждый прогон проверки сохраняется в SQLite, накапливая историю по контракту.
4. **Детектор дрейфа** — сравнивает недавнее окно успешности/латентности/стоимости с историческим базовым окном и отмечает статистически значимые регрессии.
5. **Генератор тестов** — превращает реальные провалившиеся проверки в готовые к коммиту pytest-регрессии, чтобы сбои никогда не повторялись незаметно.
6. **CI-гейт** — единая CLI-команда, которая прогоняет контракты против пакета ответов (например, из вашего eval-набора) и завершается с ненулевым кодом при превышении порогов дрейфа/бюджета, поэтому её можно встроить в любой CI-пайплайн.

## Быстрый старт

```bash
pip install -r requirements.txt

# Проверить один ответ против контракта
python -m contract_guard.cli validate examples/example_contract.yaml response.json

# Прогнать CI-гейт против пакета ответов
python -m contract_guard.cli ci-gate examples/example_contract.yaml responses.jsonl

# Сгенерировать регрессионный тест-файл из сохранённых сбоев
python -m contract_guard.cli gen-tests examples/example_contract.yaml --out tests/test_regressions.py
```

## Файлы

- `contract_guard/contract.py` — загрузка/разбор контракта (YAML/JSON -> `Contract`)
- `contract_guard/validator.py` — проверяет ответ против `Contract`
- `contract_guard/drift.py` — обнаружение дрейфа: базовое окно vs недавнее
- `contract_guard/storage.py` — история прогонов на SQLite
- `contract_guard/budget.py` — контроль бюджета по стоимости/токенам/латентности
- `contract_guard/test_generator.py` — автогенерация pytest-тестов из сбоев
- `contract_guard/ci_gate.py` — CI/CD-гейт: прогон + решение pass/fail + exit-код
- `contract_guard/reporters.py` — рендер отчётов в text/JSON/JUnit XML
- `contract_guard/config.py` — загрузка конфигурации (переменные окружения + файл)
- `contract_guard/cli.py` — точка входа командной строки
- `tests/` — юнит-тесты для валидатора, дрейфа, хранилища
- `examples/example_contract.yaml` — полностью рабочий пример контракта
